# Bug Report（增长体验升级）

## P0

1. 积分任务可“一键完成”导致验证链路失真
- 现象：发布求领养、救助参与在任务页直接点按钮即可加分。
- 风险：可绕过真实行为验证，积分体系失真。
- 处理：按钮改为“去完成”，跳转到 `publish-adopt-request` / `rescue-board`。

2. 公益捐赠点击即扣分，缺少二次确认与留言
- 现象：任务页点“立即捐赠”直接执行 RPC。
- 风险：误触成本高，缺少运营反馈闭环。
- 处理：新增独立捐赠详情页，包含留言板、捐赠方式、反馈建议入口。

3. 消息批量选择缺少“标记已读”
- 现象：编辑模式只有删除，没有状态标记动作。
- 风险：会话管理效率低。
- 处理：新增批量标记已读按钮，并同步本地 unreadCount。

4. 对话页加号功能项无响应
- 现象：点击拍照/相册/文件/位置仅关闭面板。
- 风险：用户认为功能损坏。
- 处理：接入隐藏输入控件与位置占位插入，给出明确反馈。

5. 实名认证页 dark mode 适配不足
- 现象：暗色模式下仍存在亮底表单/文本对比不足。
- 风险：可读性和一致性下降。
- 处理：补齐容器、卡片、表单 dark 样式。

## P1

1. 编辑资料弹层视觉层级偏弱
- 现象：信息块层次不明显，操作区与内容区混杂。
- 处理：重构为分组卡片、头像强调区、底部粘性操作栏。

2. 智能体入口硬编码数量有限
- 现象：消息页写死两项。
- 处理：改为配置化渲染并扩展 6 个智能体。
